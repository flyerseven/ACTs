🔧

ACT

  • 安全检查 + 白名单
  • before_action 钩子
  • 异步/同步执行
  • 超时 + 重试
  • after_action 钩子
💭

REFLECT

Reflector
  • 重复工具调用检测
  • 偏离目标评分
  • LLM 策略审查
  • stuck → 终止
✅

DONE

AgentEngine
  • LLM输出包含"DONE"
  • 或无工具调用
  • StateManager.stop("done")
  • Observer 生成报告

6. 关键设计要点
📦 数据流单向闭环

AgentEngine 是唯一的调度入口。goal 注入 → 循环运转 → AgentState 输出。每轮迭代产生一个 Step 并 add_step 到 StateManager。

⚡ LLM 决策信号

LLM 通过两个信号控制循环：输出 DONE 终止，或返回 tool_calls 触发 ACT。二者互斥，不会同时出现。

🛡 多层安全护栏

50步硬上限 | 20步防死循环 | 连续3次LLM错误终止 | 工具白名单 | before/after钩子 | 同一错误5次终止 | 10步注入停止提示。

📡 可观测性内建

Observer 贯穿全流程：Event 驱动 (start/step_start/step_end/tool_call/tool_result/reflection/done/stopped)，支持外部回调注册，自动生成 Mermaid 流程图和运行报告。
