架构总览 — 模块化 Agent 决策引擎

方案 B：各组件独立，接口清晰，决策透明

核心组件（6 个模块）

agent_engine/ 目录结构
agent_engine/
├── engine.py       ← DecisionLoop 主循环（观察→思考→行动→反思）
├── state.py        ← StateManager 状态跟踪
├── tools.py        ← ToolRegistry 工具注册/调用/验证
├── memory.py       ← MemoryManager 短期记忆 + 摘要压缩
├── reflect.py      ← Reflector 反思与自我修正
├── observe.py      ← Observer 日志/回调/Mermaid/指标
├── safety.py       ← 安全检查：步数限制/白名单/钩子
├── llm.py          ← LLM 适配器（内置 OpenAI + 自定义回调）
├── types.py        ← 共享 Pydantic 模型
├── config.py       ← 全局配置
└── builtin_tools/
    ├── search.py     ← 网络搜索
    ├── calculator.py  ← 计算器
    ├── files.py       ← 文件读写
    └── code_exec.py   ← 代码执行

关键设计决策

设计点 决策 理由
组件通信 直接方法调用，不引入事件总线 简单透明，调用链可追踪
状态持久化 可选，State.to_dict() / from_dict() 库不关心存储，由外部决定
LLM 调用 async 优先，同步为 thin wrapper 不阻塞，支持并发工具调用
工具执行 async 主路径，同步工具自动包装 兼容两种工具定义
配置 pydantic-settings + .env 文件 类型安全 + 环境变量覆盖
